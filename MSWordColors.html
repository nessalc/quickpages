MS Word Colors

Microsoft Word isn't always straightforward with its colors when it comes to examining them in code. In fact, if you pick the "wrong" color, it can be downright confusing. So I put together this guide. This may or may not apply to other Office products; I haven't confirmed that at the time of writing.

The "problem" seems down to the Theme colors, which it makes easy to choose, especially in shading the cells of tables. One nice feature of this is that you can change the theme colors and, like CSS stylesheets, it affects all documents that use theme colors.

The table below is constructed from the default theme that comes "out-of-the-box" with Word, or what it resets to if you wipe your Normal.dotm file. 60/70 of these colors may differ if you change the theme or alter the color scheme.

The first line of each entry in the table is the "name" of the color in the drop-down color picker. The second is what you get if you ask for the "value" of that color. wdColor is supposed to be a 24-bit color, at least, sort of. The third line is the associated HTML RGB color.

It's worth noting that, because of endianness, the RGB values in the last line are the reverse of the bytes from the second line.
